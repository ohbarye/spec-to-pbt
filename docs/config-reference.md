# Config Reference

This document describes every field available in `*_pbt_config.rb`, generated by `--stateful --with-config`.

## Overview

The config file is a plain Ruby Hash assigned to a constant (e.g. `StackPbtConfig`). It is user-owned and survives scaffold regeneration.

Fields are classified as:

- **Required** — the scaffold will not produce useful results without these
- **Optional** — use when the default behavior is not enough

## Quick Start

Search for `TODO(required)` in the generated config and fill in at least these three:

```ruby
StackPbtConfig = {
  sut_factory: -> { Stack.new },                      # required
  command_mappings: {
    push_adds_element: { method: :push },              # required (per command)
    pop_removes_element: { method: :pop }              # required (per command)
  },
  verify_context: {
    state_reader: ->(sut) { sut.to_a }                # required
  }
}
```

## Top-Level Fields

### `sut_factory` (required)

A lambda that creates a fresh instance of the system under test (SUT).

```ruby
sut_factory: -> { MyApp::Stack.new }
```

Called once per test run to initialize the SUT.

### `initial_state` (optional)

Override the inferred initial model state. Use when the generator's default does not match your SUT's initial state.

```ruby
initial_state: { balance: 0, limit: 100 }
```

### `command_mappings` (required)

A Hash mapping command names (symbols) to per-command configuration. Each command corresponds to a predicate in the spec.

```ruby
command_mappings: {
  push_adds_element: { method: :push },
  pop_removes_element: { method: :pop }
}
```

### `verify_context` (required)

A Hash containing verification settings.

```ruby
verify_context: {
  state_reader: ->(sut) { sut.to_a }
}
```

#### `verify_context.state_reader` (required)

A lambda that reads the observable state from the SUT. Without this, the scaffold cannot compare SUT state against the model — only model-internal checks (size changes, etc.) will run.

```ruby
state_reader: ->(sut) { sut.to_a }
state_reader: ->(sut) { { balance: sut.balance, limit: sut.limit } }
```

### `before_run` (optional)

A lambda called before each command execution.

```ruby
before_run: ->(sut) { sut.reset_counters }
```

### `after_run` (optional)

A lambda called after each command execution.

```ruby
after_run: ->(sut, result) { log(result) }
```

## Per-Command Fields

Each entry in `command_mappings` can contain:

### `method` (required)

The real Ruby method name on the SUT. The generated default is the spec predicate name (e.g. `:push_adds_element`), which is almost never the correct method name.

```ruby
push_adds_element: { method: :push }
```

### `arg_adapter` (optional)

Transform arguments before passing to the SUT. Use when the SUT API expects a different argument shape than the model.

```ruby
arg_adapter: ->(args) { [args, :option_a] }
```

### `model_arg_adapter` (optional)

Transform arguments for the model side. When only `model_arg_adapter` is set, it is also used as the default `arg_adapter` so model and SUT arguments stay aligned.

```ruby
model_arg_adapter: ->(args) { args.abs }
```

### `result_adapter` (optional)

Normalize the SUT's return value before it reaches `verify!`.

```ruby
result_adapter: ->(result) { result.to_i }
```

### `arguments_override` (optional)

Replace the default argument generator. Accepts a 0-arity or 1-arity (state-aware) lambda returning a `Pbt` arbitrary. Useful for invalid-path coverage or custom distributions.

```ruby
arguments_override: ->(state) { Pbt.integer(min: state[:balance] + 1, max: state[:balance] + 100) }
```

### `applicable_override` (optional)

Override the command's applicability check. Use for unsupported guards, richer domain preconditions, or to deliberately drive invalid calls.

```ruby
applicable_override: ->(state, args = nil) { state[:status] == :active }
```

### `next_state_override` (optional)

Override the model's state transition. Use when the inferred transition is not enough (e.g. derived state, invalid-path model changes).

```ruby
next_state_override: ->(state, args) { state.merge(balance: state[:balance] - args) }
```

### `verify_override` (optional)

Override the verification logic for this command. Receives keyword arguments including `observed_state:` (when `state_reader` is configured), `guard_failed:`, and `guard_failure_policy:`.

Leave unset when `state_reader` already exposes model-shaped state and the default equality check is sufficient.

```ruby
verify_override: ->(after_state:, observed_state:, **) {
  raise "mismatch" unless observed_state == after_state
}
```

### `guard_failure_policy` (optional)

How to handle guard-failing calls. Only relevant for commands with inferred guards.

| Value | Behavior |
|-------|----------|
| `:no_op` | Assert unchanged model and observed state |
| `:raise` | Assert the SUT raises an exception, state unchanged |
| `:custom` | Delegate to `verify_override` for domain-specific handling |

```ruby
guard_failure_policy: :raise
```
